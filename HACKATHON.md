## Формулировка (по официальной постановке)

```
A (конвейер → накопитель)  →  классификация  →  захват + перекладка  →  B | C | D
```

| Категория | Точка | Смысл |
|-----------|-------|--------|
| Подходит для сортировки | **B** | Основной сортировщик |
| Не подходит по габаритам | **C** | Негабарит |
| Не подходит без доупаковки | **D** | «Круг в сечении» → доупаковка |

Классификация: **габариты** (10×10×2 … 450×320×320 мм) → при успехе **форма** (круг в сечении, порог 0,7) → иначе категория «подходит».

Полный текст: **[task_3.md](task_3.md)** · схема участка: **[docs/LAYOUT_ABCD.md](docs/LAYOUT_ABCD.md)**.

## Архитектура (целевая)

```
FrameSource → детекция/локализация → Classifier (габариты + круг) → category
    → RoutingTable (category → zone_b/c/d) → TimingController → Actuator
```

Конфиг зон: **`config/routes.yaml`**.

Демо-fallback до классификатора: `by_class` (примитивы box/sphere/bag → B/C/D).
