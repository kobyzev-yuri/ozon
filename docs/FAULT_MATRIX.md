# Матрица сбоев: симулятор vs видео vs коррекция

> Полная карта отказов для **программного эмулятора (PyBullet)** и **реального видео**.  
> Для каждого сбоя: можно ли воспроизвести в симе, кто чинит — **детерминированные правила (WCS/WMS)**, **пересчёт ПЛК**, **LLM-арбитр** или **оператор**.  
> См. также: [ERROR_CASES.md](ERROR_CASES.md), [BUSINESS_RULES.md](BUSINESS_RULES.md).

---

## 1. Три слоя ответственности (напоминание)

```
ИДЕНТИФИКАЦИЯ (SCAN)  →  МАРШРУТ (WMS)  →  ИСПОЛНЕНИЕ (WCS/ПЛК/актуатор)
     штрихкод + CV            куда                 когда + толчок
```

**LLM-арбитр** живёт только на этапе **идентификации / маршрута** (спорный scan).  
**Блок коррекции** (детерминированный) — в основном **трекинг, ETA, отмена команд, дедуп**.  
Механические сбои (пушер, проскальзывание) **LLM не чинит** — только датчики, пересчёт, калибровка.

## 2. Сводная таблица: кто что лечит

| Тип коррекции | Что делает | Где в коде / план |
|---------------|------------|-------------------|
| **WMS-правила** | Приоритет штрихкод → кластер → CV → reject | `routing_table.py`, `routes.yaml` |
| **Индукция / фильтры** | Не сортируем шум, слипшиеся bbox | `induction.py` |
| **Повтор decode** | Другой препроцессинг crop | `barcode_decoder.py` (дальше) |
| **Голосование CV** | Класс по нескольким кадрам до scan | план: `class_vote` |
| **LLM-арбитр** | Crop + metadata → зона + reasoning | `llm_arbitrator.py`, SCAN only |
| **Привязка по barcode** | Маршрут не от `track_id`, а от кода | план: post-scan binding |
| **Пересчёт ETA** | velocity, проскальзывание | `position_tracker.py`, план P6 |
| **Отмена divert** | Трек потерян → не толкать | план P1 |
| **Триггер по геометрии** | ACTUATION LINE, не только таймер | план P2 |
| **Дедуп barcode** | Два scan одного кода | план P3 |
| **Оператор** | No Read, reject-рукав | `zone_reject` |

## 3. Идентификация (SCAN LINE)

| # | Сбой | Симулятор | Видео | Уже в коде | Эмуляция в PyBullet | Детерминированная коррекция | LLM-арбитр |
|---|------|-----------|-------|------------|---------------------|----------------------------|------------|
| I1 | **No Read** — нет штрихкода и нет класса | ✓ fallback CV/reject | ✓ блик, размытие | `no_read`, `zone_reject` | выключить `barcode_sim` + неизвестный класс | дообучить YOLO; повтор decode; расширить `routes` | **Слабо** — только если объект виден: «reject / повтор», не выдумывать EAN |
| I2 | **Ошибочный штрихкод** (460↔461) | ✓ `barcode_sim.misread` | ✓ повреждённая этикетка | `barcode_misread`, `barcode_truth` | `misread_probability` | checksum; второй скан; политика `on_conflict` | **Да** — при `barcode_cv_conflict` или низком conf; сверка crop с маршрутом |
| I3 | **Конфликт штрихкод ↔ CV** | ✓ misread + разные формы | ✓ | `barcode_cv_conflict` | комбинация misread + box/sphere | WMS: barcode > CV (прод) | **Да** — основной кейс арбитра |
| I4 | **Низкий confidence YOLO** | ✓ мятая «форма» | ✓ | `should_arbitrate` conf < 0.55 | снизить conf в детекторе / сложные объекты | голосование по кадрам; seg | **Да** |
| I5 | **Неверный класс CV** (box→sphere) | ✓ ошибка YOLO | ✓ | route через CV fallback | YOLO11n на кубах/сферах | fine-tune; ground-truth в симе только для метрик | **Да**, если нет надёжного штрихкода |
| I6 | **pyzbar / libzbar недоступен** | ✓ | ✓ | тихий fallback | `barcode_sim` при включении | установить libzbar | **Нет** |
| I7 | **Префикс не в WMS** | ✓ | ✓ | `route_source: reject` | добавить левый префикс в генератор | обновить `routes.yaml` | **Нет** |
| I8 | **Симулированный код** (не с пикселей) | только сим | — | `barcode_simulated: true` | `barcode_sim.enabled` | на проде не применимо | **Да** для демо спорных кейсов |
| I9 | **Поздний scan** (ещё `NEW`) | ✓ быстрая лента | ✓ | нет `scanned` → `ai_missed` | уменьшить `interval_steps` | `min_track_length`, SCAN правее | **Нет** |
| I10 | **Двойной scan** одной посылки | редко | ✓ | `_scanned` per track_id | — | дедуп по `barcode` (P3) | **Нет** |

### Когда выступает LLM на идентификации

```
                    ┌─────────────────────────────────────┐
  scanned           │  preliminary route (WMS + CV)       │
  preliminary ─────►│  should_arbitrate?                  │
                    │    • conf < 0.55                    │
                    │    • barcode_cv_conflict            │
                    └──────────────┬──────────────────────┘
                                   │ да
                                   ▼
                    ┌─────────────────────────────────────┐
                    │  LLM + crop → zone + reasoning        │
                    │  logs/arbitrator.jsonl                │
                    └─────────────────────────────────────┘
```

**Не вызывать арбитра:** чистый No Read без объекта в crop; механические ошибки после `scheduled`; каждый кадр (rate limit).

## 4. Трекинг (до SCAN)

| # | Сбой | Симулятор | Видео | Уже в коде | Эмуляция | Детерминированная коррекция | LLM |
|---|------|-----------|-------|------------|----------|----------------------------|-----|
| T1 | **ID-switch** (`track_id` сменился) | редко (`body_id` стабилен) | ✓ часто | ByteTrack, induction | намеренно: окклюзия в видео | после scan — маршрут по **barcode**; Re-ID; DeepSORT | **Нет** |
| T2 | **Слипшиеся объекты** (cluttering) | ✓ близкий `y_offset` | ✓ | `min_bbox_gap_px` | уменьшить зазор в spawner | индукция; YOLO-seg; два bbox | **Нет** |
| T3 | **Ложный детект** | color fallback (2D) | ✓ | `min_track_length` | — | порог conf, индукция | **Нет** |
| T4 | **Потеря трека** до scan | редко | ✓ | — | — | больше кадров до SCAN | **Нет** |
| T5 | **Неверный match YOLO↔body** | ✓ только PyBullet | — | `body_matcher` | два объекта рядом на ленте | жёстче `max_distance_px`; приоритет world X | **Нет** |
| T6 | **Прыжок bbox / jitter** | ✓ | ✓ | velocity в tracker | — | сглаживание; медиана позиции | **Нет** |

**Вывод:** трекинг чинится **инженерно**, не LLM. После успешного scan истина — **штрихкод в WMS**, не `track_id`.

## 8. Только симулятор (артефакты PyBullet)

| # | Артефакт | Влияние | Как не вводить в заблуждение |
|---|----------|---------|------------------------------|
| S1 | Нет EAN на текстуре | pyzbar пустой | `barcode_sim` + флаг `barcode_simulated` |
| S2 | `track_id` = `body_id` | стабильнее ByteTrack | говорить жюри честно |
| S3 | Ground-truth kind с спавнера | YOLO может ошибиться | kind для метрик, не для WMS при наличии кода |
| S4 | Идеальная top-down камера | нет бликов | на видео ТЗ — другой профиль ошибок |
| S5 | Фиксированная физика | не 1:1 с NBS | Digital Twin для логики, не сертификация |
| S6 | Случайный y на спавне | cluttering | `y_offset_range` |

## 10. Блок коррекции

Детерминированный слой **между** scan и `scheduled` / **между** `scheduled` и `diverted`.  
Диаграммы: [ARCHITECTURE.md](../ARCHITECTURE.md) §2a–2b, автомат: [EVENTS.md](EVENTS.md) §5a.

```
ScanStation → [CorrectionPolicy] → RoutingTable / отмена
CommandQueue → [PreDivertCheck] → SimActuator + fault_sim
```

| Модуль | Триггер | Действие | Статус |
|--------|---------|----------|--------|
| **LLM Arbitrator** | conf / `barcode_cv_conflict` | шаг ④ SCAN | **реализовано** |
| **fault_sim** | PyBullet demo | slip, miss, weak, overshoot | **реализовано** |
| **ConflictPolicy** | conflict | yaml: barcode \| cv \| arbitrator | частично |
| **BarcodeRetry** | no read | препроцессинг crop | план P2 |
| **EtaRecalc** | slip | пересчёт ETA | план P6 |
| **DivertCancel** | потеря трека | `divert_cancelled` | план P1 |
| **GeoTrigger** | ACT LINE | геометрический триггер | план P2 |
| **BarcodeDedup** | повтор кода | игнор второго scan | план P3 |

LLM — **ветка** коррекции маршрута, не замена ПЛК и не лечение `actuator_fault`.

## 11. LLM-арбитр: полная карта применимости

| Ситуация | Вызывать? | Почему |
|----------|-----------|--------|
| `barcode_cv_conflict` | **Да** | разные оси, нужен reasoning для аудита |
| `barcode_misread` (сим) | **Да** | демо ценности арбитра |
| conf YOLO < 0.55 | **Да** | crop + контекст |
| Чистый No Read | **Нет** | нет данных; reject |
| Проскальзывание / miss пушера | **Нет** | не зона vision |
| ID-switch | **Нет** | нужен barcode binding |
| После `scheduled` | **Нет** | поздно менять маршрут без оператора |
| API down / rate limit | **Нет** | fallback WMS preliminary |

Расширения арбитра (план):

| Триггер | Польза |
|---------|--------|
| `barcode_misread: true` в payload | явный вызов без conf |
| `class_vote_disagreement` | уже в yaml, доработать в коде |
| Два кадра crop (до/после) | стабильнее при блике |

## 12. Конфиг эмуляции сбоев (текущий)

`config/pybullet.yaml`:

```yaml
barcode_sim:          # идентификация
  misread_probability: 0.08

fault_sim:            # исполнение
  belt_slip: ...      # тайминг / промах по позиции
  actuator: ...       # miss / weak / overshoot
```

## 13. Сценарии для защиты (чеклист)

| Демо | Настройка | Что показать | Кто чинит |
|------|-----------|--------------|-----------|
| **Штрихкод главный** | `barcode_sim`, box + код 461 | `route_source: barcode` | WMS |
| **Ошибка скана** | `misread_probability: 0.3` | `barcode_misread`, неверный рукав | арбитр / оператор |
| **Арбитр** | `arbitrator.enabled` + конфликт | `arbitrator.jsonl` | LLM |
| **Промах пушера** | `miss_probability: 0.2` | `actuator_fault`, `actuator_miss` | ПЛК / механика |
| **Проскальзывание** | `belt_slip` 0.5 | объект не у линии ACT при толчке | ETA / датчик |
| **No Read** | `barcode_sim: false`, класс unknown | `no_read` | оператор |
| **Честное видео** | `main.py --video` | pyzbar на пикселях | без сим-инъекций |

## 14. Фраза для жюри

«Мы разделили сбои на **идентификацию** (штрихкод, CV, арбитр), **трекинг до scan** (ID, слипание) и **исполнение** (проскальзывание, актуатор). В PyBullet часть из них **воспроизводим параметрами** `barcode_sim` и `fault_sim` — с флагами в `events.jsonl`. LLM-арбитр подключён только там, где нужен **смысл по картинке** при спорном маршруте; механику ленты и пушера он не заменяет — это зона ПЛК и датчиков.»
